Additional Themes

Themes can be installed from Xupdate Store or from theme-developers' websites etc.

Uploading Themes

Distributed themes are mostly compressed. To use a theme, the uncompressed files must be upload into the /themes directory of your site.

The Theme folder must include theme.html and manifesto.ini.php. First download the theme and expand it on your local machine. Then upload the entire folder.

Theme Selection

Theme Management allows you to simple click the "Select" button next to the theme that you want to use as the default theme or main theme of your site.

Any theme that is "checked" in the "Control" column, will be available in the Theme Selection Block. If you allow users to see this Block, they will then be able to select their favorite theme.

Uninstall a Theme

The render system manages themes without using the database. So, to delete a theme, simply delete it's directory from the "theme" directory.

Template Management

The list of Template Sets is displayed in Template Management.

XCL comes with a default Template Set. By default, when modules are installed, their templates become part of this default set.

The default Template Set cannot be deleted nor modified. To customize the templates in your site, first 'Clone' the default template set.

The 'Clone' button is on the right, in the Control column.

Once you have a new Template Set, you should first make it the Active set by clicking the "Select" button in the Control column.

The actual template files can now be edited in the Template File Management article.

If you have a template set in tar.gz format, you can also upload this to your site through the admin interface.

Simply click on the "Upload a Template Set" link in Template Management.

Template File Management

  • The list of template files is displayed, and each file can be edited by clicking the edit icon ( a pencil ) in the rightmost column.

Theme folder structure

A theme is a collection of files (html, css, javascript) with Smarty Render Variables and Constants that are assigned from PHP (core, module or preload). These are referenced by preceding them with a dollar sign $ (like php) eg.: <{$smarty.const._MD_PROFILE_LANG_CONTROL}>
Variables assigned from within a template with the <{assign}> function are also displayed this way.
You can easily include a partial component anywhere using Smarty Render include that define the presentation layer.

Technically, themes only require four files : manifesto.ini.php, screenshot.png, style.css, theme.html.
The folder assets usually contains all the assets used in the theme, including images, stylesheets, and javascript files.
This is an example of the files and folders that are found in a typical XT theme folder structure.


xt-theme-html5
|-- assets/
|   |-- css/
|   |-- image/
|   `-- js/
|-- manifesto.ini.php
|-- screenshot.png
|-- style.css
`-- theme.html

You must place a theme in the "themes" folder of your installation. Each individual theme is contained in a directory named after the theme itself.
For example, the themes released for XCL 2.3.x are using a naming convention to enhance clarity and to clearly identify their contents.
A theme name starting with a prefix :

  • "xt-" simple theme, often relying on xLayout Helper Flexbox
  • "ui-" custom User Interface (icon set, language constants, modules templates).
  • "ui-framework-theme-name" using a CSS Framework eg.: ui-bootstrap-blog, ui-metro-magazine

Default System Themes

The "admin" and "default" theme names are reserved for the Core backend and frontend.
Core themes are all lowercase and might use an underscore (_) in the name.
A name convention is said to be a matter of dogma. So, you are free to adopt what is must convient to you and your project.
This is an example of the files and folders that are found in typical theme folder structure:


ui-default-jquery
|-- component/    # custom include partial template
|-- css/
|-- images/
|-- js/
|-- language/    # custom language constants
|-- templates/
|   |-- content/    # custom module templates
|   |-- forum/    # custom module templates
|   `-- user/    # custom module templates
|-- .gitignore
|-- .htaccess
|-- CHANGELOG.md
|-- manifesto.ini.php
|-- README.md
|-- screenshot.png
|-- style.css
|-- theme.html
`-- theme_assign.php

Layout

XCL-UI is defined with two custom HTML attributes.

layout — used on container(parent) element to set the layout of its children (column or row)

self — used on container(child) element to set size, align or order of display for element itself

column
div self="left"
div self="center"
div self="right"

row
self="size-1of2"
self="size-1of2"

rows
size-1of5 [width: 20%]
size-1of4 [width: 25%]
size-1of3 [width: 33.33%]
self=size-1of2 [width: 50%]
self=center m-n2 [margin negative 2]
Row center-justify class=shadow-out justify

use rows and columns for flexbox wrap, otherwise it will break the container!

Examples

This is an animated dialog which is useful for displaying information. The dialog window can be moved, resized and closed with the 'x' icon.


$( function() {
    $( "#dialog" ).dialog({
        autoOpen: false,
        show: {
            effect: "blind",
            duration: 500
        },
        hide: {
            effect: "blind",
            duration: 500
        }
    });
    $( "#opener" ).on( "click", function() {
        $( "#dialog" ).dialog( "open" );
    });
} );
