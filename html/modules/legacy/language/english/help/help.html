Getting Started

A kick start guide to XCL if you are running this admin dashboard for the first time.
The goal of this kick-start guide is to help new site administrators get orientated to what this position does and what responsibilities it entails. It provides the minimum amount of information needed, to get started and begin working with XCL Web Application Platform.
Other detailed instructional materials are available and will be highlighted throughout.

Read this guide!

Version

  • The XCL version 2.3.x ensures compatibility with Legacy modules, OOP or Object Oriented Programming, class abstraction and interfaces.
  • In 2017 major PHP projects started bug fixing and code refactoring to PHP 7.
  • In computer programming and software design, code refactoring is the process of restructuring existing computer code—changing the factoring—without changing its external behavior.
  • Refactoring is intended to improve the design, structure, and implementation of the software, while preserving its functionality.
  • Moreover, the design and code refactoring of XCL primarily provides backward compatibility and updates modules based on Duplicatable V3 (D3) architecture.
  • Content creation, filemanager with cloud storage management, WYSIWYG editor and X-Update to get and deploy from a simple user interface.
  • You can find out more information in Help menu link about features like wizard to install modules, related menus, localization, permissions and the theme engine.

Block Management

Block functionality is provided by the module to display a specific data.

For example, a block to display the "Main Menu" and another block to display the "User Menu".

The form fields are quite intuitive:

  • change the name of the block
  • set groups permissions
  • order of display
  • side (according to theme's columns)
  • set the target modules

Comment Management

Commenting systems, also called commenting platforms, add commenting functionality to websites (such as blogs and news sites) that post content. A staple of any commenting system is a text comment box where users can generate their own comments and leave them on a specific piece of content.

D3 modules comments are easily integrated (module preferences) and managed through D3Forum.

Image Management

The image management features solve the challenges of storing, organising and distributing images currently in-use and in your platform. Architected around a centralised storage space, an image management platform provides features that help people categorise, share and retrieve the images they need. The default Image management can be considered a specialised type of digital asset management (DAM) platform. Like a DAM, today’s image management solutions are typically built in the cloud, offering a large and secure storage space for libraries that might extend to hundreds of thousands – even millions – of assets.

XelFinder is easily integrated and extended to manage local and cloud storage services such as Google Drive, OneDrive or Dropbox.

Smilies Management

An emoticon is a short sequence of keyboard letters and symbols, usually emulating a facial expression, that complements a text message.

An array of emoticons is available in the insert emoticon popup of the WYSIWYG HTML editor.

Actions :

  • Add, Edit, Delete, Display
  • Code and Emotion description
  • Smiles Batch-Upload (tar.gz or zip)

Disable Emojis

Uncheck the option [Display] if you don't want your :) to turn into a (smilie).

XOOPSCube allowed file type(s) property allows you to specify the file types that the user can upload using the File field.

For example, if you set the values for this property as "gif, jpg, png" the user will only be able to upload GIF, JPG and PNG graphic images.
