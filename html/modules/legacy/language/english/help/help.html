Image Management

Control Panel > Settings > Image Management

Category management

Here you can create and edit categories, set the maximum file size, maximum image width and height, display and hide categories and images.

The following options are available when creating a category:

Name

A descriptive name of the category.

Permissions

  • Groups who can upload to the category.
  • Groups who can view the category.

File size

You can optionally set a maximum allowed size in kilobytes for uploaded file. Leave blank if you do not wish to impose a limit.

Example: 1024000 bytes = 1,000Kb = 1MB

Note: Servers usually also have built-in limits on the amount of data that can be uploaded via PHP at one time. The default value for this in PHP is 2 MB. If you have a need to upload very large files, then you should contact your Host or server admin and talk to them about any size limits they have in place.

Image width

Optionally set the maximum allowed width in pixes for uploaded images. Images over this setting will be automatically resized.

Image height

Optionally set the maximum allowed height in pixes for uploaded images. Images over this setting will be automatically resized

Display

Show or hide the category

Order

In the list component, you can sort by any column. Leave the value by default for the original sort order, or set a custom order with a numeric value.

Category store type

Choose either the file or the database storage scheme for your image files.

This setting cannot be changed later !

Keep in mind that if you select the database as the storage destination, image files will be stored as BLOB type data which may not be suitable for backup purposes.

Word Censor

Control Panel > Preferences > Word Censorship

This section of the Control Panel allows you to enable word censoring. It is also where you define the list of censored words.

Enable censorship?

You may enable or disable word censoring. If you select “Yes”, the system will replace any specified words in channel entries, comments, forum posts, etc. according to your preference below.

Replacement characters

You may optionally specify a word or phrase to be used when replacing censored words. For example, if you set “tisk tisk” as your replacement word, and “shucks” is in your censored list, then anytime “shucks” is used it will be replaced with “tisk tisk”. If you do not set this preference, a pound symbol will be used for each character that is censored, so “shucks” would be converted to “######“.

Words to censor

Specify a list of words to censor. Wildcards are allowed. For example, test* would censor the words “test”, “testing”, and “tester”, while *gress would censor the words “progress” and “congress”.

The profanity filter, sometimes referred to as just "filter" or "censor", is a feature typically used on forums and comments that automatically scans users' posts as they are submitted and automatically changes or censors particular words.

Enter the censored words in the textarea using a common delimiter character "|" (pipe) to separate the values.

Set the characters replacing all the censored words in the input field "censor-replace".

Version

  • In 2017 major PHP projects started bug fixing and code refactoring to PHP 7.
  • In computer programming and software design, code refactoring is the process of restructuring existing computer code—changing the factoring—without changing its external behavior.
  • Refactoring is intended to improve the design, structure, and implementation of the software, while preserving its functionality.
  • Moreover, the design and code refactoring of XCL primarily provides backward compatibility and updates modules based on Duplicatable V3 (D3) architecture.
